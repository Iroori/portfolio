SK하이닉스 SeedVDI 신규 구축 프로젝트

Horizon REST API 기반의 ~25,000대 대규모 VM 상태 수집 배치 플랫폼 구축. 인프라망 보안 정책에 맞춘 C# 런처와 Java 서버 간의 RSA+AES 하이브리드 암호화 통신 플로우, 그리고 사내 결재망 통합을 위한 SOAP / REST 인터페이스 설계 업무를 전담했습니다.

Backend C# .NET Java / Spring Boot Horizon REST API Security Design
01

VM 상태 수집 배치 시스템

HORIZON REST API · THREAD PARALLEL PROCESSING · UPSERT
⚠
기존 Event DB 폴링 방식의 한계
VMWARE Event DB 감지만 의존하다보니 이벤트 누락 시 실제 VM과 DB 간 상태 불일치(Dirty Read)가 발생. 약 25,000대를 커버하기에 구조적으로 부적합했습니다.
✓
목표: 10분 주기 실시간 통계화 보장
Horizon REST API를 활용해 전체 시스템 스냅샷을 최단 시간 내 병렬 수집하고, DB 부하를 차단하도록 분산 쓰기를 유도하는 신규 배치를 완성했습니다.
Step 01 — Target Selection & Failover
수집 대상 Horizon 동적 추적 (이중화)
DB 테이블에서 `collect_yn = 'Y'`인 행만 불러와 수집 타겟으로 삼습니다. 이중화 구성 서버는 Standby를 'N'으로 관리하여 불필요한 트래픽 낭비 및 중복을 제어합니다.
Step 02 — API Strategy
Pool 기준 분할/일괄 병렬 호출 여부 분석
Horizon VDI Pool 단위로 필터를 지정해 호출하는 게 빠른지, 한 번에 전체 목록을 당겨 메모리에서 처리하는 게 빠른지 테스트, 속도가 잘 나오는 병렬 파티셔닝 전략을 채택.
Step 03 — Parallel Dispatch
스레드 분배 및 병렬 수집
단일 호라이즌 최대 보유 VM만 ~12,000대. 각 Horizon마다 개별 독립적인 스레드 콜을 발생시켜 동시다발적으로 API 요청, 전체 소요 시간을 1/n으로 단축시킵니다.
Step 04 — Lock Mitigation
배치 전용 별도 Table (UPSERT + NOLOCK)
수집한 결과를 운영 테이블(`vm_main`)이 아닌 배치 전용 테이블(`vm_status_batch`)로 우회, 덮어쓰기(UPSERT) 하여 포탈 조회 프로시저와 Lock 충돌을 없앴습니다. `WITH (NOLOCK)` 지시어로 조회 레이턴시 문제를 해결했습니다.

아키텍처의 의의 — TLS(HTTPS) 인프라 통신과는 별개로 Application Layer 측면에서의 보안. 고정된 암호화 키를 코드나 DB에 저장하지 않습니다. 세션마다 생성되는 1회성 RSA-AES 랜덤 암복호화 절차에 의해, 스나이핑 등을 통한 네트워크 패킷 Replay Attack 환경을 완전히 무력화시킵니다.

03

사내 결재망 (SOAP & REST) 연동

SOAP WSDL PARSING · REST API CALLBACK · WORKFLOW AUTOMATION
🎯 목적

포탈 상의 VM 정책(생성 / 증설 / 연장)을 사내 전자결재 시스템 프로세스와 직접 연계하여 툴체인을 묶었습니다.

📦
SOAP 전용 분리 라이브러리 (JAR 모듈 직접 빌드)
사내 일부 결재 인프라가 레거시(SOAP)만 지원. 이에 따라 SOAP Envelope의 파싱과 네임스페이스 매핑 등 성가신 레이어를 포탈 본선 코드에서 덜어내어 단일 JAR 모듈 라이브러리 형태로 캡슐화시켰습니다. 포탈 측은 간단한 메소드 호출만 신경쓰면 되도록 분리 개발했습니다.
🔁
Webhook / Callback 비동기 자동화 라우트
기안자가 상신 시 즉시 VM 상태를 "결재 중(Pending)"으로 마킹, 타 시스템의 결재 완료/반려 등 외부 훅인 Callback 트리거가 도착하는 즉시, 사내 Job이 돌며 VM 프로세스(할당, 생성, 변경)가 자동 배포되도록 무인화했습니다.
