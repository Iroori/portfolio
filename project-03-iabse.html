IABSE Congress Incheon 2026 결제 시스템 구축

단기성 글로벌 컨퍼런스 행사를 위한 React 기반 SPA 프론트엔드와 Node.js/Express.js 기반의 백엔드 시스템을 통해 글로벌 PG사(PayGate) 트랜잭션 연동 및 이벤트 기간 기반 자동 부분 환불 로직을 설계 및 구현했습니다.

React Node.js Express.js PayGate API REST API Cron/Scheduler
01

글로벌 결제 시스템 (PayGate) 연동 구축

REACT · OPENPAYAPI · TRANSACTION LOGIC
🎯 구축 목표

국내/해외 참가자가 공존하는 국제 대회 특성상, 사용자의 국가 코드를 식별하여 국가별(Domestic/International) 가맹점 번호(MID)를 자동 분기 처리하고 원활한 신용카드 트랜잭션 흐름을 구축하는 것을 최우선으로 삼았습니다.

💳
Dynamic MID 분기 라우팅
React 단에서 글로벌 유저의 Select 국가 입력 값을 트래킹해, 결제 버튼(`PGIOForm`) Submit 시 국내용 가맹점과 해외용 가맹점 ID(MID)가 동적으로 맵핑되어 PayGate PG 서버 측에 전송되도록 구축했습니다. 이는 사용자의 결제 폼 입력 단계를 줄이고 불필요한 결제 오류를 방어합니다.
🔗
OpenPayAPI 컴포넌트 이식 및 검증
Step3Payment 컴포넌트에 OpenPayAPI 라이브러리를 React Lifecycle(`useEffect`, Callback 핸들러) 내에서 안정적으로 구동시키고, 콜백 결과물(`getPGIOresult`)을 Node.js 백엔드로 넘겨 서버사이드에서 무결성 이중 검증 구조를 수립했습니다.
02

일자별 부분 환불 스케줄러 개발

PAYGATE CANCEL API · PARTIAL REFUND ALGORITHM

행사 특성 상 취소 요청 시점에 따라 위약금 수수료율이 달라집니다. 기존 관리자가 수동으로 위약금을 계산하여 PG사에 취소 요청하던 업무를 프로그래밍하여 스케줄러로 자동화했습니다.

Step 01 / Date Calculation
이벤트 이벤트 기간 역산 알고리즘
사용자의 환불 요청 시점의 타임스탬프를 행사의 메인 이벤트 Date와 비교하여 <D-90(100%)>, <D-30(50%)> 등의 조건 분기율을 계산하는 로직을 Node 서버에 적용했습니다.
Step 02 / PayGate Cancel 연동
PayGate Cancel API 규격 전송
원결제 트랜잭션의 TID를 참고해 수수료가 차감된 부분 취소액(Partial Amount)을 산출하고 PayGate 취소 API에 POST 요청을 보내어 PG 취소를 단행합니다.
Step 03 / ACID Transaction
DB 상태 변경 및 이력 기록
결제 취소가 PG사에서 완료(`Success`) Code가 떨어지면 비로소 사내 DB 테이블의 상태값을 업데이트하고 환불 이력을 영구 보존합니다.
03

서버 스케줄링 운영 최적화

OPS ENVIRONMENT · CRON TASKS RESTRICTION

Cleanup Scheduler 제어 — 결제 완료되지 않은 더미 데이터를 지우는 DB Cleanup 크론(Cron) 작업이, 개발/운영 환경에서 동시에 돌며 서로 경합하는 장애를 극복했습니다. Node의 process.env.NODE_ENV 변수를 체크해 오직 Production 환경 에서만 클린업 스케줄러가 활성화되도록 제약 구조를 추가했습니다.

🗂 SPA & Backend

React 프론트엔드와 Node 런타임 간 명확한 API 분리와 역할 분담으로 구축.

💳 안정적인 결제

지역별 동적 MID 라우팅을 통해 사용자 경험 보완 및 승인 통과율 향상.

🤖 업무 자동화

관리자 수동 환불 처리 비용을 줄이기 위한 Time-based 자동 부분 환불 로직 탑재.
