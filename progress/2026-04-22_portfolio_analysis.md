# 포트폴리오 전반 분석 및 개선 과제 (2026-04-22)

## 1. 현재 구조 및 강점 (Strengths)

- **넓은 기술 스펙트럼과 풀스택 역량:** React 프론트엔드부터 Java/Spring Boot API 서버, C# .NET 데스크톱 클라이언트, AWS Lightsail 기반 인프라 배포까지 엔드투엔드로 시스템을 구축할 수 있는 실무력이 돋보입니다.
- **문제 해결(Troubleshooting) 기반의 설득력 있는 서술:** "단순히 이 기능을 만들었다"가 아니라 "기존의 한계가 무엇이어서(Dirty Read 방지, 보안 이슈, 다운타임 등), 어떤 기술적 선택을 했고(RSA+AES 이중 암호화, NOLOCK 분산 DB, 단일 인스턴스 구성), 그 트레이드오프는 무엇인지" 훌륭하게 녹아있습니다.
- **실전적인 AI Agent 플랫폼 이해도:** 단순 LLM API 연동을 넘어 LangChain 기반 Multi-Agent(Main+Sub) 바인딩, Tool Search를 통한 토큰 사용량 최적화 등 최신 Agentic 아키텍처에 대한 딥다이브 경험이 아주 좋습니다.

## 2. 스토리텔링 & 내러티브 개선 제안 (Narrative)

### 2-1. 자기 소개(One-liner) 브랜딩 부재
현재 `index.html` 최상단에 이름과 간단한 한 줄이 있으나, "어떤 가치를 창출하는 개발자인지" 뇌리에 박히는 카피(캐치프레이즈)가 부족합니다.
> **개선 방향:** "레거시 인프라 통합부터 AI 에이전트 플랫폼 구축까지, 시스템의 병목을 찾아 비즈니스 문제를 해결하는 풀스택 엔지니어 OOO입니다." 와 같은 명확한 포지셔닝 문구 도입.

### 2-2. Project 01(VDI AI)과 Project 04(Admin Console)의 관계성
현재 01번은 "AI 특화 애플리케이션"이고, 04번은 이를 찍어내는 "플랫폼(어드민 베이스)"입니다. 번호가 1번과 4번으로 떨어져 있어 글의 맥락이 단절될 우려가 있습니다.
> **개선 방향:** 인덱스 페이지에서 두 프로젝트를 'AI Agent Platform Series'로 시각적으로 묶어주거나, 01번 서론에 "이것은 04번 플랫폼 위에서 구현된 특화 에이전트 사례입니다"라는 연결고리(Link) 멘트를 추가해 거시적인 시스템 설계 능력을 보여줘야 합니다.

### 2-3. 비즈니스 임팩트 강화
기술적인 로직(어떻게 했나)은 매우 세밀한데 반해, "그래서 조직/사용자에게 얼마나 좋아졌나"에 대한 Business Value가 다소 건조합니다.
> **개선 방향:** 관리자 수동 환불 처리가 완전 자동화되어 운영 공수가 N시간 줄었다든지, 배치 수집 분산으로 포탈 렉 증상이 100% 해소되었다든지 등 사용자 체감 지표를 KPI 영역에 확실히 못 박아주는 것이 좋습니다.

## 3. UI/UX 및 코드 유지보수성 (Technical Debt)

### 3-1. CSS 코드 하드코딩 중복 제거
모든 `.html` 파일의 `<style>` 태그에 동일한 CSS 변수와 레이아웃, 카드 클래스가 수백 줄씩 복사/붙여넣기 되어 있습니다.
포트폴리오 자체도 하나의 웹 프로덕트입니다. 면접관(특히 프론트엔드 출신)이 소스를 열어봤을 때 유지보수성이 떨어져 보일 수 있습니다.
> **개선 방향:** 공통 스타일을 `assets/css/global.css`로 빼고, 각 파일에서 Link 태그로 불러오도록 리팩토링이 필요합니다.

### 3-2. Design System 일관성
`project-01`, `02`, `03` 별로 목적은 같으나 이름이나 미세한 픽셀, 색상이 다른 박스 클래스(hl-box, purpose-box, warn/ok box 등)가 파편화되어 있습니다.
> **개선 방향:** CSS 분리와 함께 컴포넌트의 가이드라인(Alert, Card, Timeline)을 일원화해야 합니다.

### 3-3. 반응형 뷰포트 확대
현재 모바일(max-width: 620px) 대응은 되어 있으나, 아이패드 등 태블릿 창 크기(768px ~ 1024px)에서 2단 배열(`two-col`)이나 타임라인 배치가 좁아져 답답해지는 구간 점검이 필요합니다.
