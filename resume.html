Horizon REST API 기반으로 사내 25,000대 규모의 가상 데스크톱 상태를 수집하고 통합 관리하는 VDI 포털 구축 프로젝트의 백엔드를 전담했습니다.

Problem
기존 DB 폴링 방식의 데이터 불일치 및 락(Lock) 지연
25,000대 규모 환경에서 Event DB 감지만으로는 상태가 누락되었고, 대량 데이터 갱신 시 포탈 조회 쿼리와 경합해 프레이징이 발생했습니다.
Solution
병렬 API 수집 및 배치 전용 테이블 (UPSERT + NOLOCK)
Horizon API 스레드 병렬 호출로 수집 시간을 단축하고, 수집 결과를 운영 테이블이 아닌 배치 전용 테이블에 UPSERT 했습니다. 조회 시 WITH (NOLOCK)을 적용해 락 충돌과 대기 프레이징 현상을 100% 해소했습니다.
Security Architecture
RSA-AES 하이브리드 암호화 통신 플로우
C# 클라이언트 런처와 Java 서버 간 통신 시 평문 노출과 네트워크 패킷 Replay Attack을 차단하기 위해, 일회용 RSA 키쌍과 랜덤 AES-256 대칭키를 혼합한 End-To-End 암호화 프로토콜을 직접 설계하고 구현했습니다.
