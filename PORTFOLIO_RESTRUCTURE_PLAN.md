# 성과 및 문제 해결(Troubleshooting) 중심 포트폴리오 개편안

본 기획안은 기존의 아키텍처 및 기능 나열식 구조에서 벗어나, 면접관에게 **"이 개발자가 어떤 어려운 문제를 어떻게 해결했고, 비즈니스에 어떤 이점을 가져다 주었는가?"**를 설득력 있게 보여주기 위해 작성되었습니다.

검토해보시고, `[수치 입력 필요]` 부분에 대략적인 체감 숫자나 내용을 채워주시면 바로 웹 페이지(HTML)에 반영해 드립니다.

## 2. Project 01: VDI AI 통합 솔루션

### Troubleshooting 1: 장기 대화 토큰 한계 초과 및 오류
*   **Problem:** 대화가 지속되면 Context Window Limit을 넘어 API 비용이 급증하고 서비스가 다운되는 위협.
*   **Solution (극복 과정):** 
    *   **Auto-Compact 도입**: 4분할 독립 토큰 측정 후, 임계치 초과 시 과거 대화를 LLM을 통해 자동 요약.
    *   **안전 마진(Usable Input) 구축**: LLM 응답 토큰을 선제적으로 빼둔 버퍼를 만들어 사전에 API 에러를 차단.
    *   **Fallback 방어**: 예외 발생 시 오래된 대화부터 버리는 FIFO(선입선출) 트리밍 시스템 도입.
*   **Achievement (성과):** 
    *   대화 세션 유지 가능성 증대 및 **토큰 비용 약 [수치: 예) 평균 40% 이상] 절약** 효과체감. 서비스 멈춤 결함 원천 차단.

### Troubleshooting 2: 독립된 AI 도구들의 응답 지연 (병목)
*   **Problem:** 시스템 진단, DB 조회 등 외부 인프라 연동이 늘어날수록 에이전트의 Action 응답 속도가 순차적으로 지연됨.
*   **Solution (극복 과정):**
    *   LangGraph를 활용하여 서로 의존성이 없는 다중 도구를 `Parallel Tool Calling` 체제로 전환.
    *   벤더(GPT, Claude 등)별로 다른 병렬 속성(True/False 반전 현상)을 `Model Factory` 패턴으로 추상화하여 확장성 확보.
*   **Achievement (성과):**
    *   에이전트 응답 체감 속도를 기존 대비 **[수치: 예) 2배 이상 / 수 초 이내] 개선**.

## 3. Project 02: SK하이닉스 SeedVDI 신규 구축

### Troubleshooting 1: 대규모 인프라 수집 시 DB Lock 및 스케일 아웃 한계
*   **Problem:** 2.5만 대에 육박하는 VM을 이벤트 기반으로만 폴링하다 보니, 상태 불일치(Dirty Read) 및 조회 프로시저의 타임아웃, DB Lock 발생. VMware 대상 API 페이징 부재로 메모리 누수 위험 존재.
*   **Solution (극복 과정):** 
    *   Horizon 별 완전히 독립된 스레드를 할당해 **초고속 병렬 수집 파이프라인** 가동.
    *   수집된 데이터를 배치 전용 테이블로 분리해 `UPSERT`하고, 포탈 조회 시 `WITH (NOLOCK)` 전략과 Cursor-based Paging 활용.
*   **Achievement (성과):** 
    *   스캔 주기를 평균 **15분 이내로 감축**하였고, 포탈 사용자 UI 멈춤/지연 현상 완전 해소.

### Troubleshooting 2: 하이브리드 보안 프로토콜 및 무결성 확보 단계
*   **Problem:** 포탈에서 C# 기반 클라이언트를 호출 시 민감한 VDI 접근 평문 자격증명이 노출되어 스니핑 / Replay Attack 에 취약함.
*   **Solution (극복 과정):** 
    *   서버에서 무작위 AES-256 생성 ➔ VDI 데이터 암호화 ➔ 생성된 AES 키를 클라이언트의 일회용 RSA-OAEP 퍼블릭 키로 Key Wrapping (이중 암호화).
*   **Achievement (성과):** 
    *   DB에 영구 암호키를 일절 남기지 않는 상태(Stateless)로 **보안 검증 무사 통과**, 제로트러스트형 보안 VDI 접속망 완성.

## 4. Project 03: IABSE 2026 참가 결제 시스템

### Troubleshooting 1: 복잡한 글로벌 결제 라우팅으로 인한 결제 이탈율
*   **Problem:** 국제 학술 행사로서 동일 페이지 내에서 내/외국인의 접속 상점ID(MID)가 분리되어야 함.
*   **Solution (극복 과정):** 
    *   React 프론트엔드 단에서 국가 코드를 식별하여, PG(PayGate) 전송 Payload 구성을 자동으로 동적 분기 처리하도록 파이프라인 통합 설계를 주도.
*   **Achievement (성과):** 
    *   오결제로 인한 문의 발생 방지 및 매끄러운 단일 UX 결제 경험(SPA) 제공.

### Troubleshooting 2: 결제 취소 프로세스의 휴먼 에러 비용 및 수동 운영 리소스
*   **Problem:** 학회 특성 상, 취소 요청 D-Day 조건(D-90은 100%, D-30은 50% 등)이 가변적이나 매번 담당자가 엑셀로 계산하여 1건씩 수동 취소를 넣어야 하는 치명적 비효율성.
*   **Solution (극복 과정):** 
    *   Time-based 환불 처리 알고리즘 적용, 수수료율 자동 산출 후 PayGate Cancel API를 트리거하는 Node.js 기반 Cron 스케줄러 배포.
    *   환경 분기 처리를 통해 개발환경/실서버 간의 스케줄러 경합을 방지(Production-Only).
*   **Achievement (성과):** 
    *   CS 담당자의 수동 취소처리 업무 시간을 **[수치: 예) 매주 약 10시간 이상] 절감하고, 정산 무결성 100% 확보**.
